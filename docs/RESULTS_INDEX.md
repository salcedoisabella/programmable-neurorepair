# Results Index

## Purpose

This document explains how to navigate the results layer of **Programmable Neurorepair**.

The project has grown through multiple technical and conceptual phases. As a result, the `results/` directory contains much more than isolated outputs. It contains the historical build path of the system.

This index is meant to help readers understand:

- how the project evolved across phases
- what each major block of results accomplished
- which layers are now closed
- which outputs matter most for understanding the current system

## How to read the results layer

The results should not be read as a flat sequence of unrelated files.

They are better understood as a build history across five major layers:

1. early engine build and state-structure derivation
2. controller refinement and boundary logic
3. documentation integration, biological re-entry, and external validation
4. intervention translation, platform framing, and externalization
5. audience-specific packaging

The most important thing to understand is that the later layers do not replace the earlier ones. They build on top of them.

## Current high-level reading

The results layer now supports a project that includes:

- a repair-state map
- a dominant repair-compatible basin
- intermediate / review-like structure
- boundary / frontier logic
- a minimal push-anchor-support architecture
- an integrated multi-stack system
- first-pass external support
- intervention-facing programs
- a validation-preparation ladder
- reviewer-facing, founder-facing, and application-specific packaging

## Phase 1 — Early engine build and state-structure derivation

### Approximate scope
Early engine steps through the first major derivation phase.

### Core purpose
This phase established the project’s initial state-structure logic.

### What this phase built
- the first repair-oriented state map
- state-transition framing within the remyelination wedge
- early candidate control interpretation
- the conceptual basis for moving beyond a descriptive atlas

### Why this phase matters
Without this phase, nothing later would exist.  
This is where the project first became more than a raw single-cell readout.

### How to read it now
Read this phase as the **foundational derivation layer**, not as the final project form.
