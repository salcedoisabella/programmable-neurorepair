# Project Overview

## Project

**Programmable Neurorepair** is a computational wedge into neural state control.

It begins with remyelination and oligodendrocyte-lineage repair, but the deeper aim is broader: to build a usable framework for identifying neural state structure, understanding transition logic, and deriving minimal control architectures that can move systems toward more functional states.

## Why this project exists

Modern neuroscience can measure cells, circuits, and molecular states at extraordinary resolution, but it still lacks a usable framework for understanding which state transitions matter, what governs them, and where those transitions become tractable for intervention.

That creates a persistent gap between **description** and **control**. We can often describe dysfunction or recovery in detail without understanding how to identify the relevant state structure, interpret the directionality of transitions, or derive minimal control logic that could guide systems toward better states.

Programmable Neurorepair exists to help close that gap.

## Why remyelination is the first wedge

This project starts with remyelination not because repair is the final destination, but because remyelination is one of the clearest biologically structured entry points into a broader problem.

Myelin is not just a repair target. It is a fundamental determinant of neural function: conduction timing, synchrony, signal propagation, circuit efficiency, and the stability of large-scale brain networks all depend on it. That makes remyelination unusually important, because it is one of the clearest places where a repair-oriented intervention could eventually connect to something larger: preserving cognition, improving resilience, restoring function more completely, and, over time, opening a path toward meaningful cognitive improvement.

In that sense, remyelination is the **first wedge**, not the endpoint.

## Core thesis

The deeper thesis of the project is that neuroscience needs to move beyond static description toward a framework that can:

- identify distinct functional state structure
- resolve the logic of transitions between states
- derive minimal control architectures from that structure
- organize intervention-facing programs around those architectures
- prioritize what should be validated first

Repair is the beginning, not the destination. The current wedge is remyelination, but the broader direction is neural state control.

## What now exists

The project now includes:

- a repair-state map
- a dominant repair-compatible basin
- intermediate / review-like structure
- boundary / frontier logic
- a minimal push-anchor-support architecture
- an integrated stack system
- first-pass external support
- an intervention-facing program layer
- a ranked validation-preparation ladder
- reviewer-facing, founder-facing, and application-specific packaging

## Minimal architecture

The current minimal architecture is:

- **Tspan2 = push**
- **Ptprd = anchor**
- **Aspa = support**

This should be read as the current best minimal control logic organizing the state-transition structure observed so far, not as a final mechanistic proof.

## State structure

The system no longer reads as a simple binary healthy-vs-diseased map.

The current interpretation includes:

- a **dominant repair-compatible basin**
- an **intermediate / review-like zone**
- a **boundary / frontier band**
- an **alternative structured basin**

This matters because it means the project is not only identifying markers of repair, but building a more explicit reading of how state structure and transition logic may actually be organized.

## Integrated system logic

The project now behaves like a multi-layer system rather than a single descriptive analysis.

### 1. State layer
Identify repair-relevant state structure.

### 2. Transition layer
Interpret intermediate, review-like, and frontier states rather than forcing a simple binary classification.

### 3. Architecture layer
Derive minimal control logic from the observed structure.

### 4. Program layer
Translate the architecture into intervention-facing programs.

### 5. Validation layer
Prioritize which programs should be tested first and define what meaningful signal would look like.

## Current stack modes

The integrated stack system currently includes four operating modes:

- **default_safe_stack**  
  Conservative interpretation and reinforcement of the dominant repair-compatible basin.

- **balanced_analysis_stack**  
  Strongest current compromise between movement and stability for intermediate states.

- **boundary_probe_stack**  
  Used to interrogate ambiguity-rich frontier behavior and transition logic.

- **maximum_rescue_probe_stack**  
  Most aggressive rescue-oriented mode; informative, but not the default operating mode.

## External validation status

External validation now exists in a first meaningful form.

### Dataset B
Dataset B currently provides the strongest first-pass external support.

It supports:

- architecture portability in first pass
- dominant basin portability in first pass
- review / boundary structure portability in first pass
- integrated stack portability in first pass

### Dataset C
Dataset C currently functions as **supportive mixed triangulation**.

It reinforces parts of the broader external story, but it is not treated as a co-equal benchmark and does not replace Dataset B.

## Intervention-facing layer

The current intervention-facing program layer includes:

- **default_safe_translation_program**
- **balanced_state_transition_program**
- **boundary_probe_program**
- **maximum_rescue_program**

These are not meant to be read as finished therapeutic claims. They are structured intervention-facing programs derived from the current state map, minimal architecture, and external support layers.

## Validation-preparation ladder

The current validation-preparation order is:

1. **default_safe_translation_program**
2. **balanced_state_transition_program**
3. **boundary_probe_program**
4. **maximum_rescue_program**

This ranking is intentional. Conservative dominant-basin reinforcement should be validated before more aggressive rescue-oriented logic.

## Guardrails

This project should be read ambitiously, but also carefully.

### What it does support
- a serious first wedge into neural state control
- a coherent state-to-architecture-to-programs framework
- meaningful first-pass external support
- a real intervention-facing and validation-facing logic layer

### What it does not yet support
- proof of general neural state control
- proof of therapeutic efficacy
- a finished therapy
- a finished company or product
- a claim that all ambiguity has already been resolved

## Long-term direction

The long-term direction is broader than remyelination alone.

This project starts with repair because repair provides a tractable first wedge. But the deeper ambition is to move toward a broader framework for:

- richer recovery
- stronger resilience
- cognition-relevant improvement
- and, eventually, more serious forms of neural state guidance and human enhancement

That long-term direction remains future-facing. The current project should be understood as the first real wedge into it.

## Naming note

**Programmable Neurorepair** is the current working name. It reflects the first wedge, not necessarily the final scope. As the project expands beyond neurorepair toward a broader neural state control framework, the name may evolve to better match the full long-term direction.
