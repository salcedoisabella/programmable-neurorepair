# Docs Home

## Purpose

This document is the main entry point for the `docs/` layer of **Programmable Neurorepair**.

The repository now contains multiple documentation layers because the project no longer behaves like a single descriptive analysis. It now includes a state map, control architecture, stack logic, external validation, intervention-facing programs, and audience-specific framing.

This page explains how to navigate those layers.

## What this documentation is for

The docs layer exists to make the project readable at different levels of depth.

Some readers need a fast overview.  
Some want the system logic.  
Some want the intervention layer.  
Some want the platform thesis.  
Some want the reviewer-facing or founder-facing interpretation.

This page helps route each reader to the right place.

## Recommended reading order

If you are new to the project, the best reading order is:

1. `README.md`
2. `docs/PROJECT_OVERVIEW.md`
3. `docs/PLATFORM_THESIS.md`
4. `docs/SYSTEM_STACKS.md`
5. `docs/INTERVENTION_LAYER.md`
6. `docs/EXTERNAL_VALIDATION.md`
7. `docs/REVIEWER_SUMMARY.md` or `docs/FOUNDER_SUMMARY.md`
8. `docs/RESULTS_INDEX.md`
