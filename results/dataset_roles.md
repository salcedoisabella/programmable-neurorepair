# Dataset Roles

Programmable Neurorepair currently uses multiple datasets with distinct roles in the computational framework.

## Dataset A
**Role:** primary derivation and state-structure sandbox

Used for:
- repair-state structure derivation
- oligodendrocyte-lineage modeling
- mature repair-state signature development
- early candidate prioritization

## Dataset B
**Role:** independent transfer-oriented validation

Used for:
- testing directional consistency in a distinct biological context
- evaluating whether the repair-oriented framework generalizes beyond the derivation setting
- supporting perturbation-relevant validation logic

## Dataset C
**Role:** WT recovery-order support

Used for:
- biological consistency support
- recovery-order alignment
- additional support for the broader repair-state progression framework

## Why these roles matter

The use of multiple datasets with different roles is important because it moves the project beyond a single-dataset descriptive analysis. The framework is being built to derive, test, and support repair-state logic across distinct contexts rather than relying on one isolated source of signal.
