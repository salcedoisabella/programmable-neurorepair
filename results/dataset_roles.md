# Dataset Roles

## Purpose

This file defines the current role of each dataset inside the Programmable Neurorepair framework.

The project is intentionally not built as a single-dataset descriptive analysis. Different datasets currently play different roles in derivation, validation, and biological support so that the framework can begin separating signal discovery from cross-context consistency.

---

## Dataset A

**Role:** primary derivation and state-structure dataset

### Current uses
- oligodendrocyte-lineage state mapping
- precursor-like vs mature repair-state structure
- mature repair-state signature derivation
- early candidate control-layer discovery
- transition-oriented modeling foundation

### Why Dataset A matters
Dataset A functions as the main computational sandbox in which the repair-state framework is derived. It is the place where the current state logic of the engine is first made operational.

---

## Dataset B

**Role:** independent transfer-oriented validation dataset

### Current uses
- directional consistency testing
- perturbation-relevant transfer support
- candidate-layer stability assessment in a distinct biological context
- early validation that the framework is not only dataset-specific

### Why Dataset B matters
Dataset B is important because it tests whether the structure derived from the main dataset can retain meaning in a separate context. This moves the project beyond single-dataset pattern fitting and toward early cross-context validity.

---

## Dataset C

**Role:** WT recovery-order support dataset

### Current uses
- biological recovery-order support
- consistency checks for repair-state ordering
- additional support for the broader remyelination progression framework

### Why Dataset C matters
Dataset C helps support the biological plausibility of the broader state-transition logic by adding recovery-order structure under WT conditions.

---

## Why these dataset roles matter

The use of multiple datasets with distinct roles is one of the most important operational features of the current engine.

It means the framework is already being built to:
- derive state structure
- test directional consistency
- support biological ordering
- move beyond one isolated source of signal

That is essential if the project is going to become a serious platform for modeling neural state transitions rather than a one-off analysis.
